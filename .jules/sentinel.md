## 2024-04-11 - DOM-based XSS via innerHTML in image fallback
**Vulnerability:** A DOM-based Cross-Site Scripting (XSS) vulnerability was found in `VaultAssetsCompact.tsx` where user-controlled input (`token.symbol`) was concatenated directly into an HTML string and assigned to `parentElement.innerHTML` during an image load failure fallback.
**Learning:** Even seemingly harmless fields like `token.symbol` can be controlled by users in decentralized applications and used for XSS payloads. Additionally, replacing `innerHTML` destroys the existing DOM nodes (including the `target` itself), so caching the parent element reference is necessary before performing the replacement and using `textContent` for safe injection.
**Prevention:** Avoid string concatenation with `innerHTML`. Define the static DOM structure via `innerHTML` using placeholder elements (with class names, not IDs, if inside a loop), cache the parent reference, and then use `textContent` on the placeholder to safely inject dynamic user input.
