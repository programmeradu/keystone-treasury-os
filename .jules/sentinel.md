## 2025-05-01 - Prevent Open Redirect Vulnerabilities
**Vulnerability:** User-controlled redirect URLs passed via URL parameters (e.g., `?redirect=` or `?next=`) were assigned to `window.location.href` and used in `NextResponse.redirect()` without validation. This allowed an attacker to create a crafted URL pointing to an external malicious domain.
**Learning:** `searchParams.get()` returns a string that could be an absolute URL. When redirecting users post-authentication or onboarding, failure to validate the redirect destination allows an Open Redirect attack. The attacker could redirect users to a phishing page after legitimate authentication, exploiting the trust users place in the application's domain.
**Prevention:** Always validate and sanitize user-controlled redirect URLs using a centralized utility (e.g., `sanitizeRedirect`). This function must verify that the path starts with a single `/` and does not start with `//` or `/\`, which can trick browsers into treating the path as an absolute domain. If the URL is not a safe relative path, it should fallback to a safe default.
