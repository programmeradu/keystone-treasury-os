## 2024-05-27 - [DOM-based XSS via Dynamic Error Messages in iframe]
**Vulnerability:** A DOM-based XSS vulnerability was found in the error rendering block of `src/components/studio/StudioToolbar.tsx`. The error message, which can contain user-controlled data or un-escaped output resulting from dynamic code evaluation or transpilation via Babel, was injected into the HTML using `innerHTML` concatenation (`'<div...><strong>Error:</strong> ' + e.message + '</div>'`).
**Learning:** Using string concatenation combined with `innerHTML` opens up a significant attack surface for XSS, especially when reflecting runtime errors from a code execution/evaluation environment. Even if `e.message` might appear generally benign from a static language perspective, an attacker controlling the runtime input can inject HTML or script tags leading to arbitrary execution inside the sandbox iframe and potentially break the expected boundaries.
**Prevention:** To prevent DOM-based XSS when injecting dynamic error messages or data into generated HTML/script strings, avoid string concatenation with `innerHTML`. Instead, define the static structure via `innerHTML` with target placeholder elements (e.g., `<span id="err-msg"></span>`), and then safely set the dynamic content using `.textContent` on the targeted element (`document.getElementById('err-msg').textContent = e.message;`).
