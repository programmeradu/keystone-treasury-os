## 2026-04-08 - DOM-Based XSS in React Error Handlers
**Vulnerability:** Found a DOM-based Cross-Site Scripting (XSS) vulnerability in `src/components/dashboard/VaultAssetsCompact.tsx`. The `onError` handler of an `<img>` tag modified `target.parentElement!.innerHTML` with an unsanitized variable (`token.symbol`), allowing arbitrary HTML/JavaScript execution if an attacker controls the token symbol.
**Learning:** Even within modern frameworks like React that usually escape content, directly manipulating the DOM using `innerHTML` with unsanitized dynamic data bypasses these protections. A secondary challenge occurs when replacing the innerHTML: the target element itself is removed from the DOM, causing subsequent references like `target.parentElement` to evaluate to `null`.
**Prevention:** Avoid string concatenation with `innerHTML`. Instead, define the static structure using `innerHTML` with placeholder elements (using classes, not IDs, to avoid duplicate IDs in lists), and then set the dynamic content securely using `textContent` on the targeted placeholder. Always cache parent element references before making destructive DOM updates.
