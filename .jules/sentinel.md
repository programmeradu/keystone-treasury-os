## 2024-05-15 - DOM-based XSS via innerHTML in React Components
**Vulnerability:** A DOM-based XSS vulnerability was found in `src/components/dashboard/VaultAssetsCompact.tsx` where an image's `onError` handler injected dynamic content (`token.symbol`) directly into a string that was then assigned to `parentElement.innerHTML`.
**Learning:** Even in React applications, manually attaching event handlers (like `onError` on native elements or interacting directly with the DOM via `e.target`) can bypass React's built-in XSS protections if string concatenation is used with `innerHTML`. Additionally, modifying `innerHTML` on the parent element removes the current target (the image) from the DOM, requiring the parent reference to be cached *before* modification to safely target the newly injected placeholder elements.
**Prevention:** Avoid string concatenation with `innerHTML` when injecting dynamic user-controlled data. Always define static structure with `innerHTML` (using class-based placeholders rather than IDs inside loops) and safely set dynamic content using `textContent` on the targeted placeholder element.
