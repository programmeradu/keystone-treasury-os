## 2024-05-18 - Path Traversal and Command Injection Mitigation in Contracts Compiler
**Vulnerability:** The API endpoint `src/app/api/studio/compile-contract/route.ts` suffered from path traversal where malicious code could provide `files` with keys like `../etc/passwd` to write outside the intended build directory because they were directly concatenated with `path.join`. Also `programName` was not validated and directly used in shell execution strings, presenting an injection risk.
**Learning:** Functions dealing with dynamic file writing via user payload must use boundary checking (`startsWith`) combined with `path.resolve`. Relying on `path.join` implicitly trusts the directory navigation (`..`). Using `child_process.exec` exposes the system to command injection through shell parsing; parameterizing arguments natively is necessary.
**Prevention:** Always validate file paths with regex constraints (e.g., `^[a-zA-Z0-9_-]+$`) and restrict resolved paths using `startsWith(baseDir + path.sep)`. To avoid command injection, enforce arguments via `child_process.execFile` instead of concatenating strings for `child_process.exec`.
