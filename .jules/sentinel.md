## 2025-04-26 - Prevent Open Redirect Vulnerability
**Vulnerability:** User-provided redirect URLs from query parameters (`searchParams.get('redirect')` and `searchParams.get('next')`) were being directly used in `window.location.href = ...` and `NextResponse.redirect(...)` without validation. This allows attackers to craft phishing links (e.g. `?redirect=https://evil.com`) that automatically redirect users to malicious sites after authentication.
**Learning:** Next.js applications frequently handle post-authentication redirects using URL parameters. Unvalidated redirects represent a critical security vulnerability and must be sanitized to ensure the target is a relative path or a known safe domain.
**Prevention:** Always validate and sanitize redirect URLs from untrusted sources (like query parameters) before using them. A utility function like `sanitizeRedirect` that enforces a relative path (e.g. starts with `/` but not `//` or `/\`) should be applied globally for all redirect parameters.
