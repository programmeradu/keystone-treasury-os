## 2024-05-15 - DOM-based XSS via innerHTML in React component event handlers
**Vulnerability:** A DOM-based Cross-Site Scripting (XSS) vulnerability was found in `src/components/dashboard/VaultAssetsCompact.tsx`. The component used string concatenation (`+ token.symbol +`) within an `innerHTML` assignment inside an `onError` event handler.
**Learning:** Using `innerHTML` with unsanitized data (even simple properties like `token.symbol`) inside an event handler is dangerous and can lead to XSS. Also, when modifying `innerHTML` of a parent element from a child's event handler (e.g., `target.parentElement!.innerHTML = ...`), the child is removed from the DOM, making subsequent `target.parentElement` references evaluate to `null`.
**Prevention:** Avoid string concatenation with `innerHTML`. Instead, define the static structure via `innerHTML` and set dynamic content using `textContent` on a targeted placeholder element. Also, always cache the parent reference (e.g., `const parent = target.parentElement!`) before modification to safely interact with newly injected elements.
