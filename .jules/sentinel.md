## 2024-05-24 - [Insecure Environment Variable Fallbacks]
**Vulnerability:** Found hardcoded fallback strings for sensitive secrets (e.g., `process.env.JWT_SECRET || 'keystone_sovereign_os_2026'`) used directly in code logic to bypass errors when the environment variables are not set.
**Learning:** This pattern is insecure because it allows the application to run using predictable, hardcoded secrets if the environment variables are accidentally omitted or misconfigured, leading to potential authentication bypass or secret compromise. In a Next.js environment, developers sometimes use these fallbacks to prevent build failures on CI where runtime secrets are unavailable.
**Prevention:** Remove hardcoded fallbacks for sensitive environment variables. Instead, explicitly check for their presence and throw an error or return a 500 status code if they are missing. If a placeholder is strictly necessary to prevent Next.js build failures on CI, conditionally use it only during the build phase by checking `process.env.npm_lifecycle_event === 'build'` or `process.env.NEXT_PHASE === 'phase-production-build'`. Never use these placeholders during production runtime.
