## 2024-04-07 - DOM-based XSS via innerHTML Concatenation in React Fallbacks
**Vulnerability:** DOM-based Cross-Site Scripting (XSS) vulnerability found in `src/components/dashboard/VaultAssetsCompact.tsx` where user-controlled token symbols were concatenated directly into a string and injected into the DOM via `innerHTML` within an `onError` image fallback handler.
**Learning:** Developers often use `innerHTML` for quick dynamic DOM structure creation in error fallback handlers without realizing that direct string concatenation bypasses React's built-in XSS protection (which normally sanitizes text content). Additionally, modifying a parent's `innerHTML` removes the child element (e.g., `e.target`) from the DOM, causing subsequent references to `target.parentElement` to evaluate to `null`.
**Prevention:** To prevent DOM-based XSS, never use string concatenation with `innerHTML`. Instead, define the static HTML structure using `innerHTML` with a placeholder (e.g., using a specific class name), and then securely inject dynamic content by querying the placeholder and setting its `textContent`. Always cache the parent element reference before modifying its `innerHTML` to avoid null reference errors.
