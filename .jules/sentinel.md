## 2024-05-24 - DOM-based XSS via innerHTML in Error Rendering
**Vulnerability:** Found multiple instances where `error.message` was directly concatenated into `innerHTML` strings inside iframe `srcDoc` generators (e.g., in `ForesightPreview.tsx`, `LivePreview.tsx`, `StudioToolbar.tsx`, and CLI `dev.ts`). This allows arbitrary JavaScript execution (DOM-based XSS) if an attacker can control the error message thrown during the runtime boot process.
**Learning:** Even internal error messages can become XSS vectors if they reflect user-supplied code or external state (e.g., when a user's code throws a crafted error string). Using `innerHTML` with unsanitized interpolation within an iframe generation context bypasses many standard React protections.
**Prevention:** Avoid using `innerHTML` with string concatenation. Instead, construct the static DOM structure using `innerHTML` and dynamically set user-controlled content using `textContent` on targeted elements (e.g., via `getElementById` or a created wrapper node).
