## 2024-03-24 - [Fix hardcoded Supabase secrets]
**Vulnerability:** Hardcoded API keys for Supabase exist in `src/lib/supabase.ts`, acting as insecure fallbacks for missing environment variables. This bypasses secure failures and allows the application to proceed with default dummy credentials.
**Learning:** These fallback keys (ending in `.XYZ`) were intentionally placed to prevent breaking Next.js static generation during builds (e.g., when real environment variables might not be present). However, having this fallback logic in production environments creates a vulnerability by allowing the app to attempt operations without valid configuration, rather than failing fast.
**Prevention:** Rather than directly returning a hardcoded dummy value if an env var is missing, create a security helper like `getEnvOrDummy(envKey, dummyValue)`. This helper should enforce secure failures by throwing an error if the key is missing in production, but explicitly bypass this check and return the dummy value ONLY if `process.env.NODE_ENV === 'test'` or `process.env.CI === 'true'` to satisfy the Next.js static build requirements without compromising runtime security.
