## 2025-03-01 - DOM Reference Issue & XSS Vulnerability in Child Event Handlers
**Vulnerability:** A cross-site scripting (XSS) vulnerability was found in `VaultAssetsCompact.tsx` image `onError` handler. The token symbol (`token.symbol`), which comes from user input/external APIs, was directly concatenated into an HTML string and injected via `innerHTML` into the parent element. Furthermore, a DOM reference issue was identified: when modifying the `innerHTML` of `target.parentElement`, the `target` itself is destroyed, causing subsequent accesses to throw an error if one were to try to use it again.
**Learning:** Directly concatenating untrusted input into `innerHTML` is an XSS vector. Modifying a parent's `innerHTML` removes its current children, invalidating the original DOM references such as `target`.
**Prevention:** Avoid injecting raw input directly into `innerHTML`. First, cache the parent element (`const parent = target.parentElement!`). Set the HTML structure (with placeholders but no variables) via `innerHTML`. Find the placeholder element inside the parent using `parent.querySelector`, and inject the dynamic string into it using the safe `.textContent` property.
