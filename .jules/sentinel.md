## 2024-05-24 - Hardcoded JWT Secret Fallbacks
**Vulnerability:** Several auth-related files contain a hardcoded fallback value for `JWT_SECRET` (`'keystone_sovereign_os_2026'`) that could be used in production if the environment variable is not set.
**Learning:** Hardcoding sensitive secrets is a critical security vulnerability. Even as a fallback, it completely compromises the authentication mechanism if an attacker discovers the static value. The application must fail-fast instead of defaulting to a known insecure key. The only exception is during tests or CI builds where Next.js requires these variables to be defined for static compilation, in which case we should use a dummy value with `.XYZ`.
**Prevention:** Remove all hardcoded fallbacks for `JWT_SECRET`. If `process.env.JWT_SECRET` is missing in production, throw a clear error or ensure the process crashes early to alert the developer. For CI/test compatibility, safely return a dummy token only when `process.env.NODE_ENV === 'test'` or `process.env.CI` is true.
