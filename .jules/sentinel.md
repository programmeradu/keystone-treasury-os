## 2024-04-06 - DOM-Based XSS via innerHTML in React Components
**Vulnerability:** Found multiple instances where dynamic variables (such as token symbols or error messages) were injected directly into the DOM using string concatenation with `innerHTML` (e.g., `element.innerHTML = '<div>' + err.message + '</div>'`).
**Learning:** This exposes the application to DOM-based Cross-Site Scripting (XSS) if the injected dynamic data is maliciously crafted. React elements generated programmatically or via direct DOM manipulation bypass React's standard HTML escaping. Additionally, modifying a parent's `innerHTML` removes existing child elements, making subsequent references to the parent element evaluate to `null` if not cached before the modification.
**Prevention:** Avoid string concatenation with `innerHTML`. Always define the static HTML structure using `innerHTML` and set dynamic data via `textContent` on a targeted placeholder element (e.g., `<span class="err-placeholder"></span>`). Always cache parent element references before modifying their HTML.
