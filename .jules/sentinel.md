## 2024-05-24 - [CRITICAL] Fix broken token generation
**Vulnerability:** The `generateToken` function in `src/app/api/alerts/subscribe/route.ts` used a flawed custom random string generator. Specifically, an inner variable declaration (`let token = ''`) inside a nested loop shadowed the outer `token` variable. This resulted in the outer `token` always being an empty string (`""`), breaking the randomness completely and allowing an attacker to trivially guess or bypass token verification.
**Learning:** Custom implementations of random string generation or cryptography often introduce critical flaws due to simple oversights (like variable shadowing or improper state management). The code incorrectly managed entropy from `crypto.getRandomValues`.
**Prevention:** Rely on well-tested, standard library functions for generating secure random identifiers, such as `globalThis.crypto.randomUUID()`. Avoid writing custom cryptography or pseudo-random generation logic unless absolutely necessary.
