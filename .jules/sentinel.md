## 2024-04-16 - Open Redirect in Authentication Redirects
**Vulnerability:** The application was directly taking `redirect` and `next` URLs from query parameters (`searchParams.get()`) without validation and redirecting users to those paths in `src/app/auth/page.tsx` and `src/app/api/auth/callback/route.ts`. This allowed an attacker to craft a malicious URL like `/auth?redirect=//evil.com` or `/api/auth/callback?next=//evil.com` which would bypass basic `/` validation and redirect the user to a malicious site.
**Learning:** React components (`window.location.href`) and server components (`NextResponse.redirect`) will follow absolute URLs or protocol-relative URLs (`//example.com`) if provided. Trusting user-provided redirect URLs from search parameters without strict validation opens up phishing attacks.
**Prevention:** Always validate and sanitize user-provided redirect paths. Ensure the URL is strictly a relative path that begins with `/` and not a protocol-relative URL (`//`), using validation logic such as `redirect.startsWith('/') && !redirect.startsWith('//')` before using it.
