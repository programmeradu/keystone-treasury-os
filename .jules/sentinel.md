## 2024-05-20 - Path Traversal Vulnerability in Dynamic Contract Compilation
**Vulnerability:** In `src/app/api/studio/compile-contract/route.ts`, an API endpoint accepted a dynamic `programName` and an object `files` mapping filenames to contents from the client request. It used `path.join` to determine destination paths and created files without validating if the resolved path stayed within the intended directory. This allowed arbitrary file write vulnerabilities via path traversal payloads (e.g., `../../../../etc/passwd`).
**Learning:** Even internal build tools or temporary artifact generation logic must strictly validate filenames. `path.join` does not prevent path traversal natively.
**Prevention:** Always validate user-supplied file and directory names (e.g., `^[a-zA-Z0-9_-]+$`). When dealing with dynamic user-supplied paths, compute the absolute path using `path.resolve()` and perform a bounds check with `startsWith(baseDir + path.sep)` to ensure the resolved file destination stays within the designated sandbox.
