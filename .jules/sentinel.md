## 2024-05-24 - DOM-based XSS via innerHTML in React/Iframe rendering
**Vulnerability:** Found multiple instances where error messages and dynamic token symbols were injected directly into the DOM using `innerHTML` string concatenation (e.g., `document.getElementById('root').innerHTML = '... ' + err.message + ' ...'`). This could allow an attacker to execute arbitrary JavaScript if they can control the error message or token symbol.
**Learning:** Even within isolated `srcDoc` iframes used for rendering code, or inside `onError` image handlers, string concatenation with `innerHTML` opens up critical XSS vectors. Developers sometimes use `innerHTML` as a shortcut to inject a quick layout block, without considering that the concatenated variables might contain unescaped HTML tags or scripts.
**Prevention:** To mitigate DOM-based XSS when injecting dynamic content into generated HTML strings, avoid string concatenation with `innerHTML`. Instead, define the static structure via `innerHTML` and set dynamic content using `textContent` on a targeted placeholder element (e.g., `<span id="err-msg"></span>` and then `document.getElementById('err-msg').textContent = err.message`). In `onError` handlers, use `document.createElement` and `appendChild` combined with `textContent`. Note: if injecting structure inside a mapping loop, use class names instead of `id` attributes for the placeholder to prevent duplicate DOM IDs.
