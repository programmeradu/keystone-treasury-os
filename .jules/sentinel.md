## YYYY-MM-DD - Command Injection Risk in Local Anchor Compile
**Vulnerability:** Command injection vulnerability in `src/app/api/studio/compile-contract/route.ts` where unsanitized `programName` input could be passed directly into bash command executions (`execAsync("anchor build", { cwd: tmpDir, ... })`) if it was used in the command, or via directory traversal in path joining (`path.join(..., programName, ...)`).
**Learning:** `programName` is derived directly from user input in the API route (`const { files, programName = "keystone_app", useCloud = false } = body;`). It's passed to `compileLocal` and used in `path.join`. If `programName` is `../../etc` it leads to path traversal. Additionally, it could cause issues with how `cargo` / `anchor` commands handle filenames. It's crucial to strictly sanitize variables used in filesystem paths and shell executions.
**Prevention:** Always sanitize user-provided inputs used in file paths (e.g., using a regex whitelist like `/^[a-zA-Z0-9_-]+$/`) to prevent directory traversal, and avoid using `exec` entirely when possible, preferring `execFile` or `spawn` with an array of arguments to prevent shell interpolation.
