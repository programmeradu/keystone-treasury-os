## 2024-04-27 - Open Redirect in Next.js Query Parameters
**Vulnerability:** The application was directly using user-supplied query parameters (e.g., `searchParams.get('redirect')` and `searchParams.get('next')`) to redirect the user via `window.location.href` and `NextResponse.redirect()` without validation. This allowed an attacker to create a link like `/auth?redirect=//evil.com` which would bypass basic relative path assumptions and perform an open redirect to `evil.com`.
**Learning:** Next.js and browser APIs like `window.location.href` allow protocol-relative URLs starting with `//` or `/\` to redirect to external domains. Just falling back to `/app` via `|| '/app'` is insufficient because it doesn't validate the provided parameter.
**Prevention:** Always validate and sanitize redirect URLs derived from query parameters. Use a centralized utility like `sanitizeRedirect` that ensures the URL is a relative path starting with `/` and explicitly rejects URLs starting with `//` or `/\`.
